Final Solutions / Easier Data Analysis

The Problem

Due to varying technical maturity of users, there is a gap where verifying data slows down users' workflow. The question became: How might we improve the application of CAQH data by improving the data analysis experience?

Key Discovery

Through our research we initially thought easier data analysis was the core need, but interviews revealed that easier data retrieval is the real gap.

Non-technical Users

Technical Users

Data analysts who need complex query capabilities and bulk data operations

Our Solution

We integrated a large language model (LLM) directly into the CAQH portal to support users such as credentialing representatives and data analysts.

Integrated Approach

We positioned the LLM not as a replacement but as a complementary tool to traditional workflows, ensuring all users can access data efficiently regardless of technical expertise.

üîó Integrated, not disruptive

Users can continue employing familiar methods like filtering and manual search. The LLM is available when they need a faster, more intuitive option.

üéØ Dual-method support

Users appreciated maintaining the ability to filter for multiple providers, a staple of their existing processes, while also leveraging LLMs for quick, conversational queries.

üí° Self-evident functionality

The system's capabilities are designed to be intuitive, reducing reliance on user initiative to figure out optimal workflows.

‚ö° Workflow streamlining

By simplifying both data retrieval and optional analysis within the same interface, productivity increases without departing from established user habits.
